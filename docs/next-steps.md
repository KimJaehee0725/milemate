# 다음 단계 계획

## 현재까지 정리
- 프로젝트 주제 초안 수립 완료
- 도메인 선정 기준 정의 완료
- 도메인 후보 비교 및 1차 추천 완료
- git 저장소 초기화 및 문서 버전 관리 시작

## 추천 의사결정 순서

### 1. 도메인 확정
가장 먼저 아래 둘 중 하나를 고른다.
- 교육기술
- 헬스케어

### 2. 세부 문제 정의
도메인 안에서 아래처럼 2~3개의 세부 문제를 뽑는다.

교육기술 예시
- AI 학습코치 서비스
- 자동 서술형 피드백 도구
- 학생 이탈 예측 및 개입 추천 시스템

헬스케어 예시
- 복약 순응도 관리 서비스
- 재활 운동 자세 교정 서비스
- 고령자 낙상 위험 조기 감지 서비스

### 3. 에이전트 출력 포맷 정의
최소 출력 포맷은 아래 6개 섹션으로 고정한다.
1. 아이디어 재정의
2. 핵심 사용자와 사용 맥락
3. 필요한 기술 요소 분해
4. 개발 난이도 평가
5. 핵심 난제와 병목
6. 우회 전략 및 MVP 경로

### 4. 자료 수집 범위 정의
도메인 확정 후 다음 자료를 수집한다.
- 논문 및 서베이
- 산업 사례 및 경쟁 서비스
- 기술 블로그와 제품 문서
- 규제 또는 운영상 제약

## 현재 추천안
### 추천안 A: 교육기술 중심
장점
- MVP와 데모가 가장 쉽다.
- 팀 내부 커뮤니케이션이 수월하다.
- 사례 수집이 빠르다.

리스크
- 기술혁신론 수업에서 임팩트가 약해 보일 수 있다.

### 추천안 B: 헬스케어 중심
장점
- 기술혁신과 시장/제도 통합을 잘 보여준다.
- 학술성과 사회적 중요성이 크다.

리스크
- 규제와 검증 문제 때문에 범위가 쉽게 커진다.

## 내 현재 판단
수업 데모와 구현 안정성을 우선하면 교육기술이 가장 적합하다.
수업에서 이론적 임팩트와 혁신성을 더 강하게 가져가고 싶다면, 범위를 좁힌 헬스케어가 좋다.

## 권장 전략
- 1차 구현 도메인은 교육기술로 잡는다.
- 발표에서는 헬스케어에도 확장 가능한 프레임워크라고 설명한다.
- 즉, 실제 MVP는 교육기술에서 검증하고, 일반화 가능성은 헬스케어 사례로 보완한다.

이 전략은 구현 안정성과 발표 설득력을 동시에 잡기 쉽다.
